# Tolgee REST API Specification

If you would like to get the OpenAPI specification in JSON format, you can download it [here](https://app.tolgee.io/v3/api-docs/Public%20API%20(All)).

If you are more familiar with Swagger UI, follow this link
[here](https://app.tolgee.io/swagger-ui/index.html?configUrl=%2Fv3%2Fapi-docs%2Fswagger-config&urls.primaryName=Public%20API%20(All)).
There, you can also find internal API specifications that are not intended for public use.

Super JWT token is required for sensitive operations when user has set 2FA. You can obtain super JWT token via `/v2/user/generate-super-token` endpoint. But be careful! Super JWT Token has super powers!

It's not recommended to use API key in query param, since it can be stored in logs.

<div><table><tbody><tr><th>Security Scheme Type:</th><td>apiKey</td></tr><tr><th>Header parameter name:</th><td>ak</td></tr></tbody></table></div></TabItem></SchemaTabs></div>
      

 - The Bearer Auth (JWT Token) is used internally by the Tolgee Platform frontend. Some of the endpoints require a `Super JWT`
token, which requires two-factor authentication and is only valid for a short period of time.
 - The project API endpoints `/v2/projects/:projectId/...` can be accessed with Project API key
 - The non-project API endpoints have to be accessed with PAT (Personal access token)

The allowed authentication methods are specified for each endpoint.
